What it is

An OS-layer for the operator's vault.

system-o is the generic, installable substrate of the operator-vault pattern — validated over months of AI-augmented development work, stripped of user-specific content, and packaged so any operator can install it fresh.

▤
Vault scaffold
Folder structure, _meta/ scaffolding, conventions, registry templates, daily-note template. Drop it on a fresh machine.
▸
Offline automation chain
Six PowerShell scripts wired to Windows Task Scheduler. Triage, purge, backup, radar, sweep, report — every night, even when the machine is locked.
◎
Agent-portability stance
Gates are local-deterministic. The LLM is a pluggable endpoint, not the policy-maker. Ships with a CLAUDE.md template you customize once.
◷
Lifecycle conventions
Handoff verification, frontmatter tiers, risk tiers, decide-by clocks, sweep windows, sewerpipe retention. All documented, all enforced by local scripts.
▦
Plugin manifest
Obsidian plugin set spec — Dataview, Templater, Kanban, Canvas. Installer resolves at setup time. No vendoring, no licensing headaches.
⬡
Dashboard views
HOME dashboard, system-map Canvas, per-project Kanban skeletons. Content-agnostic — they surface your data, not placeholder copy.
How it works

The nightly chain runs itself.

Six tasks, registered as SYSTEM-principal scheduled tasks. Fire at staggered times. Self-log. Backup before reporting. No cloud, no relay, no cron daemon.

Principles

Load-bearing constraints.

These aren't aspirations. Anything that violates them gets re-designed before it ships.

Offline-first.
The chain runs with no internet connection. Cloud is optional augmentation, never a dependency. A markdown OS that requires constant cloud isn't an OS — it's a UI.
Friction-free in, structured out.
Schema is applied by automation at routing time, not by the human at write time. Capture happens unstructured. The script applies the template on the move out of inbox.
Gates are local-deterministic.
The LLM is a participant, not the policy-maker. Destructive operations are gated by local scripts with verifiable artifacts — never by model confidence alone.
The vault plays four roles.
Operator manual the agent reads. Operator manual the agent writes. Workspace where work happens. Audit log of what happened. All four, simultaneously.
